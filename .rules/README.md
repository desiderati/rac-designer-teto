# Regras do Diretório `.rules-new`

## Objetivo

Este diretório reúne regras funcionais do produto em linguagem de negócio, para leitura de pessoas técnicas e não
técnicas.

## Como usar estes documentos

1. Ler primeiro este arquivo para entender a organização.
2. Depois abrir o tema específico que deseja validar.
3. Em discussões de produto, priorizar estas regras como referência de comportamento esperado.

## Temas disponíveis

1. `canvas.md`
    - Como funciona o espaço de desenho, seleção, navegação, atalhos e segurança de edição.

2. `toolbar.md`
    - Como menus e comandos do editor devem se comportar no uso diário.

3. `vistas-por-tipo.md`
    - Limites e regras de inserção/remoção de vistas para cada tipo de casa.

4. `piloti-nivel.md`
    - Regras de edição de nível e consistência estrutural.

5. `piloti-mestre.md`
    - Regra de mestre único e efeitos dessa escolha.

6. `contraventamento.md`
    - Fluxo de criação, remoção e restrições por coluna/lado.

7. `viewer-3d.md`
    - Regras de visualização 3D, sincronização e inserção de snapshot no 2D.

8. `refactoring-agents.md`
    - Regras operacionais dos agentes de análise e execução de refatoração.

## Princípios deste diretório

1. Linguagem clara e orientada a comportamento de produto.
2. Foco em regra funcional, não em implementação técnica.
3. Atualização contínua conforme o produto evolui.
4. Coerência entre documentos (sem regras conflitantes).

## Quando atualizar

Atualize estas regras quando houver mudança em:

1. Fluxo de interação para o usuário.
2. Limite/regra de negócio.
3. Regra de segurança operacional do editor.
4. Processo oficial dos agentes de refatoração.
